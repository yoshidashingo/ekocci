# ekocci v2 (エコちっち)

Apple Watchで育てるバーチャルペット育成ゲーム。

> **ekocci** = eco + cchi (ちっち)。エコロジーをテーマにした、手首の上の小さな生き物。

---

## Overview

エコちっちは、Apple Watch上でバーチャルペットを育てる育成ゲームです。たまごっちの核となるゲームメカニクスをベースに、Apple Watchのハードウェア特性(Digital Crown、Taptic Engine、コンプリケーション、常時表示ディスプレイ)を活かしたネイティブ体験を提供します。

**コンセプト**: 1回のインタラクションは5〜15秒。手首をチラッと見て、サッとお世話して、すぐ日常に戻れる。

---

## 1. ライフサイクル (成長段階)

ペットは以下の段階を経て成長します。

| 段階 | 名称 | 期間 | 説明 |
|------|------|------|------|
| 🥚 | **たまご** | 約5分 | 画面上でたまごが揺れ、自動的に孵化する |
| 👶 | **あかちゃん** | 約1時間 | 常にお世話が必要。ミニゲーム不可。この段階では死なない |
| 🧒 | **こども** | 約24時間 | ミニゲーム解禁。頻繁にお世話を求める |
| 🧑‍🎤 | **ヤング** | 約2〜3日 | 進化分岐が始まる。お世話品質で次の姿が変わる |
| 🧑 | **おとな** | 無期限(寿命まで) | 最終形態。キャラクターが確定する |
| 👴 | **シニア** | おとな後期 | 長寿ペットのみ到達。特別な姿に変化 |
| 💀 | **おわり** | - | 寿命またはお世話放棄で死亡。新しいたまごからやり直し |

**寿命**: お世話品質により約7〜25日間。完璧なお世話で最長寿命を達成できます。

## 2. ステータスパラメータ

ペットの状態を表す4つの主要パラメータがあります。

| パラメータ | 表示 | 説明 |
|-----------|------|------|
| **おなか** | ♥♥♥♥ (4段階) | 時間経過で減少(約60〜80分で1つ減る)。ごはんで回復 |
| **ごきげん** | ♥♥♥♥ (4段階) | 時間経過で減少(約40〜60分で1つ減る)。遊びやおやつで回復 |
| **たいじゅう** | グラム表示 | ごはん(+1g)、おやつ(+2g)で増加。ミニゲーム(-1g)で減少。太りすぎは進化に悪影響 |
| **しつけ** | 0〜100% (25%刻み) | 正しくしつけると+25%。進化時の姿に大きく影響 |

**補助パラメータ**:

| パラメータ | 説明 |
|-----------|------|
| **年齢** | 実時間24時間 = 1歳。起床時にカウントアップ |
| **世代** | 何代目のペットか。世代を重ねるとボーナスあり |
| **エコポイント** | ゲーム内通貨。ミニゲームやお世話で獲得 |

## 3. お世話アクション

### 3.1 ごはん (Feed)

2種類の食事でお世話します。

| 種類 | 効果 | 副作用 |
|------|------|--------|
| **ごはん (Meal)** | おなか +1 | たいじゅう +1g |
| **おやつ (Snack)** | ごきげん +1 | たいじゅう +2g |

- おなかが満タンでもごはんを与えられる(太るだけ)
- おやつの与えすぎは肥満 → 進化に悪影響
- キャラクターごとに好き嫌いがある(将来実装)

### 3.2 あそぶ (Play) / ミニゲーム

ごきげんを回復し、たいじゅうを減らすメインの遊び。

| ゲーム | ルール | 報酬 |
|--------|--------|------|
| **どっち? (Left or Right)** | ペットがどちらを向くか当てる。5回中3回正解で勝ち | ごきげん +1、たいじゅう -1g |
| **おおきい? ちいさい? (High or Low)** | 次の数字が大きいか小さいか当てる。5回中3回正解で勝ち | ごきげん +1、たいじゅう -1g |
| **ジャンプ! (Jump)** | 障害物をタイミングよくタップで避ける | ごきげん +1、たいじゅう -1g、エコポイント獲得 |

- Digital Crownを回してゲームを選択
- 勝つとエコポイントも獲得

### 3.3 トイレそうじ (Clean)

- ペットは定期的にうんちをする(あかちゃん: 15〜30分毎、おとな: 2〜4時間毎)
- 画面上にうんちアイコンが出現(最大4個まで蓄積)
- そうじアクションで除去
- **放置すると**: 10〜15分で病気になるリスクが上昇

### 3.4 くすり (Medicine)

- ペットが病気になるとドクロアイコンが表示される
- くすりアクションで治療(1〜3回投与が必要)
- **病気の原因**: うんち放置、お世話放棄、おやつの食べ過ぎ
- **放置すると**: 数時間で死亡

### 3.5 しつけ (Discipline)

しつけシステムはゲームの核心的なメカニクスです。

**しつけるべきタイミング**:
- ペットが呼んでいるが、おなか・ごきげん共に空ではない → **わがまま**(しつけが必要)
- ごはんを拒否する → **いたずら**(しつけが必要)

**しつけてはいけないタイミング**:
- 本当におなかが空のとき
- 本当にごきげんが低いとき
- 病気のとき
- 寝ているとき

正しいしつけ1回で **しつけメーター +25%**。進化時に100%だと最良の進化先になります。

### 3.6 でんき (Light)

- ペットが寝ると画面にZZZが表示される
- **15分以内に消灯**しないとお世話ミスにカウント
- 寝ている間はお世話不可
- 睡眠中はおなか・ごきげんの減少速度が約50%に低下

## 4. 進化 (Evolution) システム

お世話の品質によって進化先が決まります。

### 4.1 お世話ミス (Care Miss)

進化を左右する最重要の隠しパラメータ。以下の場合にカウントされます:

| お世話ミスの条件 | 猶予時間 |
|----------------|---------|
| おなかが0のまま放置 | 約15分 |
| ごきげんが0のまま放置 | 約15分 |
| 病気を放置 | 即時〜数時間 |
| うんち長時間放置 | 10〜15分 |
| 寝たのに消灯しない | 約15分 |
| わがまま呼び出しを無視 | 約15分 |

### 4.2 進化先の決定ロジック

```
お世話ミス 0〜1回 + しつけ100% → Tier 1 (最良キャラ)
お世話ミス 1〜2回 + しつけ75%  → Tier 2 (良キャラ)
お世話ミス 3〜5回 + しつけ50%  → Tier 3 (普通キャラ)
お世話ミス 6回以上             → Tier 4 (残念キャラ)
```

### 4.3 進化ツリー (例)

```
🥚 たまご
 └─ 👶 あかちゃん
     ├── 🧒 こどもA (良いお世話)
     │    ├── 🧑‍🎤 ヤングA → 🧑 おとなA (Tier 1: 最良)
     │    ├── 🧑‍🎤 ヤングA → 🧑 おとなB (Tier 2: 良)
     │    └── 🧑‍🎤 ヤングB → 🧑 おとなC (Tier 3: 普通)
     └── 🧒 こどもB (悪いお世話)
          ├── 🧑‍🎤 ヤングC → 🧑 おとなD (Tier 3: 普通)
          └── 🧑‍🎤 ヤングC → 🧑 おとなE (Tier 4: 残念)
```

- 各世代で20種類以上のキャラクターが登場予定
- 隠しキャラクターの解放条件あり

## 5. 死亡 (Death) メカニクス

### 5.1 死因

| 死因 | 条件 |
|------|------|
| **餓死** | おなか0を数時間放置 |
| **病死** | 病気を12〜24時間放置 |
| **老衰** | 寿命(完璧なお世話でも避けられない) |
| **蓄積ダメージ** | お世話ミスの蓄積で寿命が大幅に短縮 |

### 5.2 死亡後

- 死亡画面(天使が飛んでいくアニメーション)が表示される
- 最終年齢が記録される
- 「つぎのたまご」で新しいペットを開始 → 世代カウンター +1
- 世代を重ねるとボーナス(初期エコポイント増加など)

## 6. 睡眠スケジュール

キャラクターの成長段階によって就寝・起床時間が異なります。

| 段階 | 就寝 | 起床 | 睡眠時間 |
|------|------|------|---------|
| あかちゃん | 20:00 | 09:00 | 約13時間 |
| こども | 20:00〜21:00 | 09:00〜10:00 | 約12時間 |
| ヤング | 21:00〜22:00 | 09:00〜10:00 | 約11時間 |
| おとな | 22:00〜23:00 | 08:00〜09:00 | 約10時間 |

- 睡眠中はうんちが出ない
- 一部のキャラクターは夜型(就寝が遅い)
- 日中に昼寝をするキャラクターもいる

---

## 7. 呼び出し (Attention) システム

ペットがお世話を求めると通知が届きます。

### 7.1 呼び出しの種類

| トリガー | 表示 | 対応 |
|---------|------|------|
| おなかが空 | 呼び出しアイコン点灯 | ごはんを与える |
| ごきげんが空 | 呼び出しアイコン点灯 | 遊ぶ or おやつ |
| 病気 | ドクロアイコン | くすりを与える |
| うんち | うんちアイコン出現 | トイレそうじ |
| 就寝 | ZZZ表示 | でんきを消す |
| わがまま | 呼び出しアイコン点灯(メーター正常) | しつけ |

### 7.2 猶予時間

呼び出しから**約15分以内**に対応しないとお世話ミスがカウントされます。

### 7.3 サイレントニーズ(通知なし)

以下は通知されないため、定期的にチェックが必要です:
- たいじゅう(太りすぎ/痩せすぎ)
- おなか・ごきげんが低いが0ではない状態
- しつけメーターの進捗

## 8. ショップ / アイテム

エコポイントで購入できるアイテム。

### 8.1 フード

| カテゴリ | 例 | 効果 |
|---------|-----|------|
| ごはん系 | おにぎり、サラダ、カレー | おなか回復、たいじゅう微増 |
| おやつ系 | クッキー、フルーツ、アイス | ごきげん回復、たいじゅう増加 |
| 特別食 | キャラ限定メニュー | ごきげん大幅回復 |

### 8.2 おもちゃ

- ペットと遊べるアイテム(ボール、ぬいぐるみなど)
- 使用でごきげん回復
- キャラクターごとにお気に入りのおもちゃがある

### 8.3 アクセサリー

- ペットに着せ替え可能(帽子、メガネ、リボンなど)
- 見た目のカスタマイズのみ(ステータスへの影響なし)

### 8.4 部屋デコレーション

- 背景(壁紙)の変更
- 家具の配置

---

## 9. 世代 (Generation) システム

ペットが寿命で亡くなると、次の世代が始まります。

- 世代カウンターが +1 される
- 新しいたまごは前の世代の「子」として扱われる
- **世代ボーナス**: 初期エコポイント増加、レアキャラ解放条件の緩和
- **家系図**: 歴代のペットを家系図で閲覧可能

## 10. Apple Watch ネイティブ機能

### 10.1 コンプリケーション (文字盤ウィジェット)

文字盤からペットの状態を一目で確認できます。

| コンプリケーション | 表示内容 |
|------------------|---------|
| **Rectangular** | ペットのスプライト + おなか/ごきげんバー |
| **Circular** | ペットの顔 + 気分アイコン |
| **Inline** | テキスト(例: 「おなかへった!」) |
| **Corner** | おなかゲージ(円弧表示) |

- 約15分ごとに自動更新
- タップでアプリ起動

### 10.2 通知 (Notifications)

| 通知タイプ | 頻度 | Haptic |
|-----------|------|--------|
| おなか空き | おなか0になった時 | `.notification` |
| ごきげん低下 | ごきげん0になった時 | `.notification` |
| 病気 | 発症時 | `.directionDown` (緊急) |
| うんち | 出現時 | `.click` |
| 就寝 | 寝た時 | `.click` |
| 死亡危機 | 死亡まで1時間以内 | `.retry` (強い振動) |

- **1日2〜3回を上限**とし、通知疲れを防止
- 緊急度に応じてHapticの強さを変える

### 10.3 Digital Crown

| 操作 | アクション |
|------|----------|
| 回す | メニュースクロール、ミニゲーム操作 |
| 長押し | ペットをなでる(ごきげん微回復) |

### 10.4 常時表示ディスプレイ (Always-On Display)

- 常時表示モード: ペットの静止スプライト + ステータスアイコン(アニメーションなし)
- 手首を上げると: フルアニメーション + インタラクティブモード
- 低電力モード時は常時表示オフ

### 10.5 Taptic Engine

| イベント | Haptic |
|---------|--------|
| ごはんを食べた | `.success` |
| ミニゲーム勝利 | `.success` |
| ミニゲーム敗北 | `.failure` |
| しつけ成功 | `.directionUp` |
| 進化! | `.notification` + `.success` |
| 死亡 | `.stop` |

## 11. 一時停止 / 時間管理

### 11.1 おやすみモード (Pause)

- お世話できない時間帯にペットを「おあずけ」できる
- おあずけ中はすべてのパラメータ減少が停止
- 1日最大10時間まで(乱用防止)
- バッテリー切れ中もパラメータは凍結(バッテリー切れ = 死亡にはしない)

### 11.2 リアルタイムクロック

- ペットの時間は実時間と連動
- 24時間 = 1歳
- 就寝・起床は実時間に基づく
- 時計を手動で変更してもペットの内部時計は独立して管理
